Context

What Is Credit Risk?

💳 The Loan

A bank lends you money today. You promise to repay it over time — with interest on top. The bank earns interest; you get capital.

⚠️ The Risk

The borrower might default — stop paying. Banks lend billions, so even a small default rate means massive losses.

🎯 The Goal

Before lending, predict: Will this person repay? A good model lets banks charge fair rates and avoid bad loans.

Why Multivariate Statistics?

A FICO score is just one number. Default risk depends on dozens of factors interacting. Multivariate methods see the full picture.

Why does this matter?

Statistical methods assume clean, complete data. Missing values, wrong data types, and extreme outliers will produce garbage results. "Garbage in, garbage out."

Class Imbalance Handling

79% good loans vs 21% defaults. We balanced training data at 2:1 ratio (undersampling) so the model learns real patterns, not just "predict good for everything."

Interdependence Technique

Factor Analysis

🔍 What It Does

Groups many correlated variables into a smaller number of hidden (latent) factors. No outcome variable — purely about structure in the data.

💡 The Analogy

You can't directly measure "intelligence" — but you see it reflected in test scores across math, reading, and logic. Factor Analysis finds that hidden dimension automatically.

✅ Bartlett's Test — PASSED

χ² = 657,867  |  p < 0.001
Variables are significantly correlated. Factor Analysis is appropriate.

📏 Kaiser Criterion

Keep factors with eigenvalue > 1. A factor must explain more variance than a single original variable to be worth keeping.

Dependence Technique 1

Linear Discriminant Analysis (LDA)

🎯 What It Does

Finds the linear combination of variables that maximally separates two groups — Good Borrowers vs. Defaulters. Produces one "discriminant score" per person.

💡 The Intuition

Imagine all borrowers as dots in a multi-dimensional space. LDA finds the single direction where — if you projected all dots onto it — the blue (good) and red (default) groups overlap as little as possible.

📋 Assumptions

Variables roughly normally distributed within each class
Equal covariance across both classes
Linear boundary between classes

⚖️ Training Setup

15 origination features · Z-score standardized · 2:1 balanced sample · 70/30 train-test split

Dependence Technique 2

Logistic Regression

📊 What It Does

Predicts the probability of default — a number between 0% and 100%. Uses the S-shaped sigmoid function to keep outputs as probabilities. If probability > 50% → predict Default.

🆚 How It Differs from LDA

LDALogistic Regression
Assumes normalityNo assumption
Maximizes separationMaximizes likelihood
Discriminant scoreProbability (0–1)

✅ Why LR Wins Here

Financial variables are NOT normally distributed — incomes, balances, and rates are right-skewed. LR's weaker assumptions make it more robust and it outperforms LDA here.

It plots True Positive Rate (catching actual defaults) vs. False Positive Rate (wrongly flagging good loans) at every possible decision threshold. A perfect model goes straight to the top-left corner.

Critical Analysis

Key Limitation: Data Leakage

The full dataset models (97–99% accuracy) include variables that are only known AFTER the loan has been resolved. This is called data leakage — the model is cheating.

🔴
total_rec_prncp

Total principal recovered — only exists after the loan ends

🔴
recoveries

Money recovered post-default — only populated IF there was a default

🔴
collection_recovery_fee

Fee charged to recover defaulted funds — post-event only

🔴
last_pymnt_amnt

Last payment made — known only after payment history unfolds

💡 The Analogy

Trying to predict if a student will fail — and using their final exam score as one of your predictors. Of course the model is perfect. You're using the answer to predict the answer.

✅ Our Live Predictor Fixes This

The interactive predictor on the previous slide uses only 15 origination-time features — variables known at loan application. That's why its accuracy is ~71%, not 99%. That's honest and realistic.

📚 What Real Banks Do

Only use application-time data (FICO, income, DTI)
Must comply with ECOA & fair lending laws
Models are audited for discriminatory patterns
